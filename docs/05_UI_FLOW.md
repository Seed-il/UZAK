# UI Flow

## 1. 개요

UZAK MVP는 웹소설 읽기 및 간단한 연재 기능을 제공한다.

UI는 다음 원칙으로 설계한다:

- 단순하고 직관적인 구조
- 최소 클릭으로 주요 기능 접근
- 로그인 상태에 따라 기능 제한

---

## 2. 화면 구성

## 주요 페이지 목록

```text
/                      → 홈
/login                 → 로그인
/register              → 회원가입
/novels                → 작품 목록
/novels/:novelId       → 작품 상세
/chapters/:chapterId   → 회차 읽기
/write/novel           → 작품 생성
/write/chapter         → 회차 작성
````

---

## 3. 사용자 흐름 (Reader)

### 3.1 비로그인 사용자

```text
홈 접속
↓
작품 목록 조회
↓
작품 클릭
↓
작품 상세 보기
↓
회차 클릭
↓
회차 읽기
```

---

### 3.2 로그인 사용자 (독자)

```text
로그인
↓
홈 이동
↓
작품 목록
↓
작품 상세
↓
회차 읽기
```

---

## 4. 사용자 흐름 (Writer)

```text
로그인
↓
작품 생성
↓
작품 목록에서 확인
↓
회차 작성
↓
작품 상세에서 회차 확인
```

---

## 5. 페이지별 상세 흐름

---

## 5.1 홈 (/)

### 역할

* 작품 목록 진입점

### 구성 요소

* 상단: 로고 / 로그인 버튼
* 본문: 작품 리스트

### API

```text
GET /api/novels
```

---

## 5.2 로그인 (/login)

### 역할

* 사용자 인증

### 사용자 행동

```text
이메일 입력
비밀번호 입력
로그인 버튼 클릭
```

### API

```text
POST /api/auth/login
```

### 성공 흐름

```text
로그인 성공
↓
JWT 저장
↓
홈 이동
```

---

## 5.3 회원가입 (/register)

### 역할

* 신규 사용자 생성

### 사용자 행동

```text
이메일 입력
비밀번호 입력
닉네임 입력
역할 선택 (reader / writer)
회원가입 버튼 클릭
```

### API

```text
POST /api/auth/register
```

### 성공 흐름

```text
회원가입 성공
↓
로그인 페이지 이동
```

---

## 5.4 작품 목록 (/novels)

### 역할

* 전체 작품 리스트 표시

### 구성 요소

* 작품 카드 리스트
* 제목 / 설명 / 작가명

### API

```text
GET /api/novels
```

### 사용자 행동

```text
작품 클릭
↓
작품 상세 이동
```

---

## 5.5 작품 상세 (/novels/:novelId)

### 역할

* 작품 정보 및 회차 목록 표시

### 구성 요소

* 작품 제목
* 작품 설명
* 작가 정보
* 회차 리스트

### API

```text
GET /api/novels/:novelId
```

### 사용자 행동

```text
회차 클릭 → 회차 읽기
```

---

## 5.6 회차 읽기 (/chapters/:chapterId)

### 역할

* 회차 본문 읽기

### 구성 요소

* 회차 제목
* 본문 내용

### API

```text
GET /api/chapters/:chapterId
```

---

## 5.7 작품 생성 (/write/novel)

### 역할

* 새로운 작품 생성

### 접근 조건

* 로그인 필요

### 사용자 행동

```text
제목 입력
설명 입력
공개 여부 선택
생성 버튼 클릭
```

### API

```text
POST /api/novels
```

### 성공 흐름

```text
작품 생성
↓
작품 상세 페이지 이동
```

---

## 5.8 회차 작성 (/write/chapter)

### 역할

* 회차 작성

### 접근 조건

* 로그인 + 작성자

### 사용자 행동

```text
작품 선택
회차 제목 입력
본문 입력
순서 입력
작성 버튼 클릭
```

### API

```text
POST /api/novels/:novelId/chapters
```

### 성공 흐름

```text
작성 완료
↓
작품 상세 이동
```

---

## 6. 인증 상태별 UI 제어

---

## 비로그인 상태

```text
가능:
- 작품 조회
- 회차 읽기

불가능:
- 작품 생성
- 회차 작성
```

UI 처리:

* 작성 버튼 숨김
* 로그인 버튼 표시

````

---

## 로그인 상태

```text
가능:
- 작품 생성
- 회차 작성
````

UI 처리:

* 작성 버튼 활성화
* 로그아웃 버튼 표시


## 9. MVP UI 원칙

* 복잡한 UI 금지
* 최소 기능 우선
* 페이지 수 최소화
* 반응형은 이후 고려
